ToDo API - Flask приложение

Простое REST API для управления задачами (ToDo-лист).

-Быстрый старт

1. Установите зависимости:
bash
pip install flask
Запустите сервер:

python run.py
Сервер запустится на http://localhost:5000

Проверка работоспособности
Проверьте здоровье сервера:

curl http://localhost:5000/health
Ожидаемый ответ: {"status": "healthy", ...}

Создайте тестовую задачу:

curl -X POST http://localhost:5000/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "Тестовая задача", "priority": "normal"}'
Получите список задач:

curl http://localhost:5000/tasks
Основные эндпоинты
GET /tasks - получить все задачи

POST /tasks - создать задачу (требует title и priority)

GET /tasks/{id} - получить задачу по ID

PUT /tasks/{id} - обновить задачу

POST /tasks/{id}/complete - отметить как выполненную

DELETE /tasks/{id} - удалить задачу

GET /statistics - статистика по задачам

GET /health - проверка здоровья сервера

Параметры задачи
При создании/обновлении задачи требуется:

json
{
  "title": "Название задачи",
  "priority": "low|normal|high"
}
Фильтрация задач
Параметры GET запроса:

isDone=true/false - фильтр по статусу выполнения

priority=low/normal/high - фильтр по приоритету

Пример:

curl "http://localhost:5000/tasks?isDone=false&priority=high"
Тестирование
Запустите тесты:

python -m pytest test_api.py
Архитектура
app/ - основное приложение Flask

run.py - точка входа для запуска сервера

test_api.py - тесты API

tasks.json - файл хранения данных

Возможные проблемы и решения
Сервер не запускается - проверьте, что порт 5000 свободен

Ошибка импорта Flask - убедитесь, что Flask установлен

Ошибки при сохранении - проверьте права на запись файла tasks.json

Тесты не проходят - убедитесь, что сервер не запущен на порту 5000 во время тестов

Дополнительные утилиты
Для загрузки тестовых данных используйте:

python json_loader.py --create  # создать пример файла
python json_loader.py tasks.json  # загрузить задачи
Примечание: В json_loader.py измените порт с 8080 на 5000 для работы с Flask.
